       The  umask  is  used  by [1;31mopen[0m(2), [1;31mmkdir[0m(2), and other system calls that[m
       create files to modify the permissions placed on newly created files or[m
       directories.   Specifically,  permissions  in  the umask are turned off[m
       from the [1;32mmode[0m argument to [1;31mopen[0m(2) and [1;31mmkdir[0m(2).[m
[m
       Alternatively, if the parent directory has a default ACL (see  [1;31macl[0m(5)),[m
       the umask is ignored, the default ACL is inherited, the permission bits[m
[7m Manual page umask(2) line 1 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K       are set based on the inherited ACL, and permission bits absent  in  the[m
[7m Manual page umask(2) line 2 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K       [1;32mmode[0m  argument  are turned off.  For example, the following default ACL[m
[7m Manual page umask(2) line 3 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K       is equivalent to a umask of 022:[m
[7m Manual page umask(2) line 4 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K[m
[7m Manual page umask(2) line 5 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K           u::rwx,g::r-x,o::r-x[m
[7m Manual page umask(2) line 6 (press h for help or q to quit)[0m[K[K [KESCESC[KOO[KBB[K[m

       The  constants  that  should  be  used to specify [1;32mmask[0m are described in[m
       [1;31minode[0m(7).[m
[m
       The typical default value for the process  umask  is  [1;32mS_IWGRP[0m [1;32m|[0m [1;32mS_IWOTH[0m[m

[m
           S_IRUSR | S_IWUSR | S_IRGRP | S_IROTH[m

[1;31mNOTES[0m[m
       A child process created via [1;31mfork[0m(2) inherits its parent's  umask.   The[m
       umask is left unchanged by [1;31mexecve[0m(2).[m

       the same time changing it.  A second call  to  [1;31mumask[0m()  would  then  be[m
       needed  to restore the umask.  The nonatomicity of these two steps proâ€[m
       vides the potential for races in multithreaded programs.[m

       Since Linux 4.7, the umask of any process can be viewed via  the  [1;32mUmask[0m[m
       field  of [1;32m/proc/[pid]/status[0m.  Inspecting this field in [1;32m/proc/self/staâ€[0m[m
       [1;32mtus[0m allows a process to retrieve its umask without  at  the  same  time[m
       changing it.[m
